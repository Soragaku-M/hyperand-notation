I was interested in googology. Whenever I was bored in class, I'd spend some time doing googology. Little by little, I started to form a notation. Eventually, I made it far into the realm of large numbers, but I had no way of publishing it. I didn't have access to coding tools back then. I had no idea how to code back then. But now, I have those, and now, I can finally show my creation to the world.
Here, I'll try to explain how it works. Here's an example of a valid Hyperand Notation:

(25) 3A5 2A4 5A2 9A 55

Here, the (25) is the base, the 3A5 2A4 5A2 9A are the hyperands, and 55 is the mantissa. These are the 3 parts of a valid Hyperand Notation, although the base can be discarded. If there is no base displayed, it is automatically 10. Now let's define the notation. We'll use the variable b for "base" and n for "mantissa". The simplest hyperand is "A" and it is defined like this:

(b) A n = bn

Examples:
A 3 = 103 = 1,000
A 5 = 105 = 100,000
(27) A 4 = 274 = 531441
(37) A 5 = 375 = 6.934e7
(15) A 10 = 1510 = 5.766e11

These numbers are huge! Now you might be wondering if we can stack the A hyperand. You might think it would go like (b) A A n but actually that's invalid. Hyperands should be in descending order. The proper way is 2A. Here's how 2A is defined:

(b) 2A n = (b) A bn

Examples:
2A 2 = A 102 = 1e100
2A 4 = A 104 = 1e10,000
(3) 2A 15 = (3) A 315 = 3.248e6,846,168
(8) 2A 16 = (8) A 816 = ee14.405
(12) 2A 128 = (12) A 12128 = ee138.168

These numbers are astronomical! Take the 1st example, that number is about 100,000,000,000,000,000,000x bigger than the amount of atoms in the observable universe! If each universe was also an atom, then the 2nd number is equivalent to the amount of atoms in the 125th level universe. The third being the 85,571st level universe, and the fourth and the fifth having levels very large. Now you might be able to guess how 3A is defined. (b) 3A n = (b) 2A bn, and 4A is defined like this: (b) 4A n = (b) 3A bn, and 5A is defined like this: (b) 5A n = (b) 4A bn, then 6A, 7A, 8A, and so on... The general rule is:

(b) (m+1)A n = mA bn

Note: 0A is always discarded.
Example:

The next level is A3. It's defined like this:

(b) A3 n = (b) nA2 1

Like the previous levels, you can have multiple amounts of A3, and you can combine it with the lower level hyperand, as long as you put them in decreasing order.
Example:
15A3 3A2 8A 9 =
  • Hyper-E notation: E9#8#4#16
  • PsiCubed2 letter notation: H15.596
  • BEAF/BAN: [4, 16, 4]
  • Arrow notation: 10^^^^15
  • FGH: f5(14)
If you have noticed, A is exponential level, A2 is tetrational level and A3 is pentational level. The higher the hyperand level, the higher the growth level. You can probably guess what the next hyperand level is: A4, and it's definition is (b) A4 = (b) nA3 1. We can generalize this to this rule:

(b) Am n = (b) nAm-1 1

Note: any hyperand with amount 0 should be discarded

Decimalization

Currently, Hyperand notation can only have an integer mantissa, which is absolutely diabolical. So, let's fix it. The A hyperand has no problems with decimals, so let's move on to the higher level hyperands. When the mantissa is not an integer, then this applies:

(b) Am n = (b) floor(n)Am-1 bmod(n,1)

Great! Now there's nothing diabolical with my notation. Now, I will show you the formal definition of hyperand notation.
  • # represents the remainder hyperands. b, n, m, and o can mean any positive integer. A hyperand is in the form mAn. A is a shorthand for A1
  • Rule 1: (b) # 0An o = (b) # o
  • Rule 2: (b) # mA o = (b) # (m-1)A bo
  • Rule 3: (b) # mAn o = (b) # (m-1)An oAn 1
